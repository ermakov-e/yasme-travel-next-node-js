# User Flows — Пользовательские сценарии

> Этот документ описывает все пользовательские пути в MVP. Используется как карта при реализации каждого экрана и API-эндпоинта.

---

## Обзор сценариев

| UC | Сценарий | Страница | Ключевой API |
|---|---|---|---|
| UC-01 | Первый вход | `/login` → `/groups` | `GET /auth/yandex`, `GET /auth/callback` |
| UC-02 | Повторный вход (сессия жива) | `/groups` | `GET /auth/me` |
| UC-03 | Создание группы | `/groups` (модал) | `POST /groups` |
| UC-04 | Просмотр группы и галереи | `/groups/:id` | `GET /groups/:id`, `GET /groups/:id/photos` |
| UC-05 | Загрузка фото | `/groups/:id` | `POST /photos/presign`, `POST /photos/confirm` |
| UC-06 | Карта поездок | `/map` | `GET /groups` |
| UC-07 | Список друзей | `/friends` | `GET /friends` |
| UC-08 | Редактирование профиля | `/profile` | `PATCH /users/me`, `POST /storage/avatar-presign` |
| UC-09 | Выход из аккаунта | `/profile` | `POST /auth/logout` |
| UC-10 | Приглашение в существующую группу | `/groups/:id` | `POST /groups/:id/members` |

## UC-01: Первый вход (новый пользователь)

**Триггер:** Пользователь впервые открывает приложение (нет cookie `token`)

```
[Браузер]                   [Next.js Frontend]          [Fastify Backend]
    │                              │                            │
    │  Открывает /                 │                            │
    │─────────────────────────────>│                            │
    │                              │  GET /api/v1/auth/me       │
    │                              │  (cookie отсутствует)      │
    │                              │──────────────────────────> │
    │                              │  401 Unauthorized          │
    │                              │<────────────────────────── │
    │  redirect → /login           │                            │
    │<─────────────────────────────│                            │
    │                              │                            │
    │  Нажимает "Войти через Яндекс│                            │
    │  → переход на /api/auth/yandex                            │
    │──────────────────────────────────────────────────────────>│
    │                              │  302 → oauth.yandex.ru    │
    │<──────────────────────────────────────────────────────────│
    │                              │                            │
    │  [Яндекс: вход, подтверждение прав]                       │
    │                              │                            │
    │  /api/v1/auth/callback?code=XXX&state=YYY                 │
    │──────────────────────────────────────────────────────────>│
    │                              │  Upsert user, sign JWT    │
    │                              │  Set-Cookie: token=<jwt>  │
    │                              │  302 → /groups            │
    │<──────────────────────────────────────────────────────────│
    │                              │                            │
    │  /groups (пустой список)     │                            │
    │─────────────────────────────>│                            │
```

**Экран `/login`:**
- Логотип Yasme Travel + tagline
- Кнопка "Войти через Яндекс ID" (синяя, с иконкой Яндекса)
- Нет других элементов

**Экран `/groups` (первый раз, нет групп):**
- Пустое состояние: иллюстрация + "Создай первую поездку"
- FAB кнопка [+] внизу справа

**Граничные случаи:**
- Пользователь отменил OAuth → возврат на `/login` с сообщением об ошибке
- Яндекс недоступен → `toast("Ошибка входа, попробуй позже")`

**Компоненты:** `YandexLoginButton`, `LoginPage`

## UC-02: Повторный вход (сессия жива)

**Триггер:** Пользователь открывает PWA, cookie `token` валиден (< 7 дней)

```
PWA открывается
→ (app)/layout.tsx (Server Component) читает cookie
→ GET /api/v1/auth/me → 200 { user }
→ Пользователь попадает сразу на /groups
```

**Граничные случаи:**
- Cookie истёк (> 7 дней) → 401 → redirect `/login`
- Токен подделан → 401 → redirect `/login`

## UC-03: Создание группы

**Триггер:** Пользователь нажимает FAB [+] на странице `/groups`

```
/groups
  └─ FAB [+] нажата
     └─ Открывается Sheet (mobile) / Dialog (desktop)
        │
        ├─ ШАГ 1/3: Обложка + название
        │   ├─ Загрузка обложки: presign → PUT → storageKey сохраняется локально
        │   ├─ Input: название группы (required, min 2 chars)
        │   └─ Кнопка "Далее →"
        │
        ├─ ШАГ 2/3: Локация
        │   ├─ Яндекс.Карта с возможностью тапнуть для выбора точки
        │   ├─ ИЛИ: поле поиска адреса (Yandex Geocoder API)
        │   ├─ Выбранная точка показывается маркером
        │   └─ Кнопки "← Назад" и "Далее →"
        │
        └─ ШАГ 3/3: Участники
            ├─ Поле поиска по имени/телефону
            ├─ Список всех пользователей (GET /friends)
            ├─ Чекбоксы для выбора
            └─ Кнопка "Создать поездку"
               └─ POST /groups { name, coverKey, lat, lng, address, memberIds }
                  └─ Успех: Sheet закрывается, новая карточка появляется вверху списка
                  └─ Ошибка: toast с описанием
```

**API:**
1. `POST /groups/:groupId/photos/presign` (для обложки — temp key)
2. `PUT presignedUrl` (прямо на S3)
3. `POST /groups` `{ name, coverKey: "temp/...", lat, lng, address, memberIds }`
   - Бэкенд копирует `temp/...` → `groups/{id}/cover.ext`

**Анимации:**
- Sheet появляется снизу (Vaul spring physics)
- Переключение шагов: `AnimatePresence` с горизонтальным слайдом
- Индикатор прогресса: 3 точки, активная подсвечена `text-primary`

**Граничные случаи:**
- Пропуск обложки → группа создаётся с дефолтным градиентом (цвет из названия)
- Пропуск участников → только создатель (OWNER)
- Сеть отвалилась на шаге 2 загрузки → `toast("Попробуй снова")`, шаги не сбрасываются

**Компоненты:** `CreateGroupModal`, `StepCoverAndName`, `StepLocation`, `StepInviteFriends`

## UC-04: Просмотр группы и фотогалереи

**Триггер:** Тап по карточке группы на `/groups` или тап по маркеру на карте

```
/groups (или /map)
  └─ Тап по GroupCard (или MapMarker)
     └─ Переход на /groups/:id
        │
        ├─ Шапка: обложка (full-bleed), название, кнопка ←
        ├─ Аватары участников (стек, до 5)
        ├─ Masonry grid фотографий (GET /groups/:id/photos?page=1&limit=20)
        │   ├─ Skeleton-заглушки при загрузке
        │   └─ Infinite scroll (следующая страница при приближении к концу)
        └─ FAB [📷] для загрузки фото
```

**При тапе на фото (Lightbox):**
```
Тап на фото
  └─ AnimatePresence: фото расширяется на весь экран (shared layout animation)
     ├─ Имя загрузившего + дата
     ├─ Свайп влево/вправо для листания
     └─ Тап вне фото или кнопка × → закрытие (обратная анимация)
```

**API:** `GET /groups/:id`, `GET /groups/:id/photos?page=N&limit=20`

**Граничные случаи:**
- Нет фотографий → пустое состояние: "Будьте первым, кто добавит фото"
- Ошибка загрузки → `error.tsx` с кнопкой "Попробовать снова"

**Компоненты:** `GroupDetailPage`, `PhotoGallery`, `PhotoLightbox`

## UC-05: Загрузка фото

**Триггер:** Тап на FAB [📷] на странице `/groups/:id`

```
FAB [📷] нажата
  └─ <input type="file" accept="image/*" multiple> — нативный пикер
     └─ Пользователь выбирает 1–10 фото
        └─ Для каждого фото параллельно:
           1. Валидация: размер ≤ 10MB, тип image/jpeg|png|webp
           2. POST /groups/:id/photos/presign { filename, contentType, fileSize }
           3. PUT presignedUrl (прогресс-бар для каждого файла)
           4. POST /groups/:id/photos/confirm { storageKey }
        └─ По завершении: новые фото появляются вверху галереи
           (TanStack Query invalidateQueries(['photos', groupId]))
```

**UI во время загрузки:**
```
┌────────────────────────┐
│ [превью фото]          │
│ ████████░░░░  75%      │ ← прогресс-бар
│ photo_001.jpg          │
└────────────────────────┘
```

**Граничные случаи:**
- Файл > 10MB → `toast("Файл слишком большой, максимум 10 МБ")`
- Неподдерживаемый формат → `toast("Поддерживаются JPEG, PNG, WebP")`
- Ошибка сети → конкретный файл помечается красным, кнопка "Повторить"
- Частичная загрузка (5 из 8) → показать результат, предложить повторить упавшие

**Компоненты:** `PhotoUploadFAB`, `usePresignedUpload` (хук)

## UC-06: Карта поездок

**Триггер:** Тап на таб "Карта" в BottomNav

```
/map
  └─ Яндекс.Карта (весь экран, BottomNav поверх)
     ├─ Центрирование: последняя активная группа или геолокация пользователя
     ├─ Маркеры: все группы пользователя (GET /groups → lat/lng)
     │   └─ Каждый маркер: кастомная иконка с миниатюрой обложки группы
     └─ Тап на маркер:
        └─ BottomSheet появляется снизу:
           ├─ Обложка группы (aspect 16/9)
           ├─ Название + локация
           └─ Кнопка "Открыть поездку" → /groups/:id
```

**API:** `GET /groups` (возвращает lat, lng для каждой группы)

**Техническая особенность:**
- `YandexMap` загружается через `dynamic(() => import(...), { ssr: false })`
- Скрипт Яндекс.Карт подключается через `<Script strategy="lazyOnload">`
- Компонент монтируется **один раз**, скрывается/показывается через CSS, не unmount

**Граничные случаи:**
- Нет групп → карта без маркеров + подсказка "Создай поездку, чтобы увидеть её на карте"
- Яндекс.Карты не загрузились (нет сети) → placeholder с текстом об ошибке

**Компоненты:** `MapPage`, `YandexMap`, `GroupMarker`, `GroupPreviewSheet`

## UC-07: Список друзей

**Триггер:** Тап на таб "Друзья" или выбор участников при создании группы (UC-03, шаг 3)

```
/friends
  └─ Список всех зарегистрированных пользователей (GET /friends)
     ├─ Поиск по имени (клиентская фильтрация)
     ├─ Для каждого пользователя:
     │   ├─ Аватар (или инициалы)
     │   ├─ Имя
     │   └─ Номер телефона (если предоставлен Яндексом)
     └─ (MVP: нет кнопки "Добавить в друзья" — просто справочник)
```

**В контексте UC-03 (шаг 3):**
- Тот же список, но с чекбоксами
- Выбранные пользователи отображаются аватарами вверху списка

**API:** `GET /friends` (возвращает всех пользователей кроме себя)

**Граничные случаи:**
- Нет других пользователей → "Ты первый здесь! Пригласи друзей"
- Очень длинный список (500+ пользователей) → виртуализация через react-virtual (post-MVP пока, для MVP просто скролл)

**Компоненты:** `FriendsPage`, `FriendCard`, `FriendsList`

## UC-08: Редактирование профиля

**Триггер:** Тап на таб "Профиль" в BottomNav

```
/profile
  ├─ Аватар (большой, по центру)
  │   └─ Тап → <input type="file" accept="image/*">
  │              └─ presign → PUT → PATCH /users/me { avatarKey }
  │                 └─ Аватар обновляется оптимистично
  │
  ├─ Имя пользователя
  │   └─ Тап → inline input (replace text with input)
  │              └─ Потеря фокуса / Enter → PATCH /users/me { name }
  │
  ├─ Телефон (readonly, из Яндекса)
  └─ Кнопка "Выйти" → UC-09
```

**API:** `POST /storage/avatar-presign`, `PATCH /users/me`

**Граничные случаи:**
- Имя < 2 символов → inline ошибка "Слишком короткое"
- Аватар > 5MB → `toast("Выбери фото меньше 5 МБ")`
- Ошибка сохранения → откат оптимистичного обновления

**Компоненты:** `ProfilePage`, `AvatarUpload`, `InlineNameEdit`

## UC-09: Выход из аккаунта

**Триггер:** Кнопка "Выйти" на странице `/profile`

```
/profile → "Выйти" нажата
  └─ Диалог подтверждения: "Выйти из аккаунта?"
     ├─ "Отмена" → диалог закрывается
     └─ "Выйти" → POST /auth/logout
                  └─ Бэкенд очищает cookie
                     └─ Redirect на /login
```

**Граничные случаи:**
- Ошибка сети → cookie всё равно удаляется на клиенте, redirect на `/login`

## UC-10: Приглашение в существующую группу

**Триггер:** Тап на иконку участников в шапке `/groups/:id`

```
/groups/:id
  └─ Иконка участников (аватары стеком) нажата
     └─ Sheet со списком участников
        ├─ Для каждого участника: аватар + имя + роль (owner/member)
        │   └─ Владелец видит × для удаления участника
        └─ Кнопка "Пригласить"
           └─ Открывается список друзей (как UC-07, но в Sheet)
              └─ Выбирает пользователей → POST /groups/:id/members
                 └─ Toast "Пользователи добавлены" + список обновляется
```

**API:** `GET /groups/:id/members`, `POST /groups/:id/members { userIds: string[] }`

**Граничные случаи:**
- Пользователь уже в группе → он не показывается в списке для приглашения
- Нет прав (не владелец) → кнопка "Пригласить" скрыта

## Граничные состояния (общие)

| Состояние | Поведение |
|---|---|
| Нет интернета | Показывать офлайн-баннер, кэшированные данные через React Query |
| Сессия истекла | При 401 → `toast("Сессия истекла")` → redirect `/login` |
| Нет прав (403) | `toast("Нет доступа")` + не менять экран |
| 404 (группа удалена) | `not-found.tsx` → кнопка "Вернуться к списку" |
| Пустые списки | Всегда пустое состояние с иллюстрацией и CTA |
| Долгая загрузка | Skeleton-заглушки (не спиннеры) |
